Sistemas de Gerenciamento de Banco de Dados

O Sistema de Gerenciamento de Banco de Dados é o software que orquestra as operações realizadas pelo banco de dados. Ele tira a responsabilidade de manipular os dados do front-end e forma o back-end. Nos dias de hoje, os modelos mais usados são:

Bancos Relacionais

Um banco de dados relacional funciona de forma semelhante a uma planilha, onde cada coluna representa um tipo de dado e cada linha representa um conjunto de informações.

Uma tabela é composta por linhas e colunas, onde cada linha armazena os dados de um objeto, identificada por uma Primary Key (PK), que serve como um identificador único, enquanto cada coluna representa um atributo desse objeto.

Bancos Não Relacionais

SGBD Não-Relacional: Um banco de dados não-relacional funciona de maneira flexível, permitindo o armazenamento de dados em estruturas como pares “chave: valor”. Nesse modelo, cada dado é armazenado como uma chave, que serve como identificador único, e um valor, que pode ser qualquer tipo de informação associada a essa chave Ver Exemplo.

O formato JSON (JavaScript Object Notation) é muito utilizado em bancos não-relacionais. O JSON permite armazenar dados com estruturas aninhadas de “chaves”:”valores”, o que proporciona flexibilidade no armazenamento de informações. Saiba Mais.

Principais Bancos de Dados

PostgreSQL

É um banco de dados relacional com suporte a SQL. Ele é otimizado para consultas complexas e grandes volumes de dados, sendo capaz de transações ACID. O Postgre é ideal para aplicações que exigem integridade de dados e desempenho.

MySQL

É um banco de dados relacional que utiliza SQL para consultas. Ele tem ótima performance, com suporte a transações ACID. Pode apresentar limitações em operações complexas de dados em comparação com outros DBs.

MongoDB

É um banco de dados NoSQL (não-relacional) orientado a documentos. Ele é muito escalável e ideal para dados semi-estruturados ou flexíveis. Embora suporte transações ACID em versões recentes, seu foco é agilidade - com uso da Consistência Eventual.

O que é um dicionário de dados?

Um dicionário de dados é como um catálogo que descreve a organização dos dados em um banco de dados. Ele informa quais tabelas existem, o que cada coluna armazena e quais tipos de dados são usados. Esses tipos de dados são importantes porque determinam o formato das informações armazenadas em cada campo, garantindo que os dados sejam manipulados corretamente e de forma eficiente. Existem diversos tipos de dados, entre eles:

Dados Numéricos

Tipos como INT, DECIMAL e FLOAT armazenam números inteiros ou com ponto flutuante.

Dados de Texto

CHAR e VARCHAR são usados para armazenar texto, com ou sem limite fixo de caracteres.

Dados Compostos

São coleções de outros dados, onde você pode armazenar múltiplos valores juntos.

Dados de Tempo

DATE e DATETIME guardam datas e horários, permitindo manipulação precisa do tempo.

O que é Modelagem de Banco de Dados?

A Modelagem de Banco de Dados é o processo anterior à própria criação do banco de dados, onde se define a estrutura de armazenamento e as relações do banco.

Modelo DER

Um Diagrama Entidade-Relacionamento representa as entidades e relações de um banco de dados de forma gráfica. As entidades são representadas por retângulos, e as relações por linhas conectando-as.

Modelo MER

Um Modelo Entidade-Relacionamento é uma versão mais detalhada do modelo DER, que inclui informações sobre a cardinalidade das relações entre as tabelas. A cardinalidade define como as tabelas se conectam: 1:1 significa que cada registro de uma tabela se relaciona com um único registro de outra; 1:N indica que um registro de uma tabela pode se relacionar com vários registros de outra; e N:N significa que muitos registros de uma tabela podem se relacionar com muitos registros de outra.
