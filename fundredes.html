Fundamento de Redes

Redes de computadores

Redes de computadores são sistemas que conectam dois ou mais dispositivos para compartilhar informações. Elas permitem a comunicação entre computadores, smartphones, servidores e outros equipamentos, possibilitando o acesso à internet. As redes podem variar em tamanho, desde pequenas redes domésticas até grandes redes corporativas e a própria internet.

O que é uma Rede Mínima?

Uma rede mínima é a configuração simples necessária para estabelecer comunicação entre dois dispositivos. Ela consiste em pelo menos dois computadores conectados por um meio de transmissão e um protocolo de comunicação que permite a troca de dados.

O que é a Internet?

A Internet é uma rede global de computadores que permite a comunicação e o compartilhamento de informações mundialmente. Ela funciona através de protocolos padronizados (como o TCP/IPd) que permitem que dispositivos diferentes se comuniquem. A Internet surgiu na década de 1960 como um projeto militar e evoluiu para se tornar a maior rede de comunicação do planeta, conectando bilhões de dispositivos e pessoas.

O que a compõe?

A Internet é uma infraestrutura complexa. Fisicamente, inclui cabos submarinos, cabos de fibra óptica terrestres, satélites, roteadores, servidores e data centers distribuídos pelo mundo. Logicamente, é composta por protocolos de comunicação (TCP/IP, HTTP, DNS), sistemas de endereçamento (endereços IP), provedores de serviços de internet (ISPs) e milhões de redes menores interconectadas. Toda essa estrutura trabalha em conjunto para garantir que os dados cheguem do remetente ao destinatário.

O que é a Web?

A Web (World Wide Web) é um sistema de informação que funciona sobre a Internet, composto por páginas interligadas através de hiperlinks.

Web e Internet não são a mesma coisa: a Internet é a infraestrutura de rede, enquanto a Web é um serviço que funciona sobre essa infraestrutura.

O Mapa global de Fibra Óptica

O mapa global de fibra óptica mostra a rede de cabos submarinos e terrestres que, basicamente, forma a Internet. Esses cabos, transmitem dados com fibras opticas muito pequenas, conectando paises e continentes.

O Hardware de uma Rede Wi-Fi

Uma rede Wi-Fi é composta por diversos componentes que trabalham juntos para fornecer conectividade sem fio. A qualidade da conexão depende da frequência utilizada (2.4GHz ou 5GHz), do padrão Wi-Fi e de fatores como distância e obstáculos.

Roteador

Dispositivo que transmite o sinal Wi-Fi e gerencia o tráfego de dados entre dispositivos conectados e a Internet.

Access Point

Dispositivo que estende a cobertura da rede Wi-Fi, conectando-se ao roteador principal via cabo para criar áreas adicionais de cobertura sem fio em ambientes maiores.

Repetidor/Extensor

Equipamento que capta o sinal Wi-Fi existente e o retransmite, ampliando o alcance da rede sem necessidade de cabos adicionais entre os dispositivos.

Hardware de uma Rede Local

Uma rede local (LAN) utiliza componentes físicos para conectar dispositivos através de cabos de rede. Essa estrutura oferece maior estabilidade, velocidade e segurança comparada às redes wireless.

Switch

Equipamento que conecta múltiplos dispositivos em uma rede local, gerenciando o tráfego de dados e direcionando pacotes especificamente para o dispositivo de destino correto.

Cabos Ethernet

Cabos (Cat5e, Cat6, Cat6a) que transmitem dados entre dispositivos. Quanto maior a categoria, maior a velocidade e menor a interferência.

Placa de Rede

Interface de hardware instalada nos computadores que permite a conexão física à rede através de cabos Ethernet, convertendo dados em sinais elétricos.

Servidor

Computador dedicado que fornece recursos, serviços e armazena dados compartilhados para todos os dispositivos conectados na rede local.

O que são Procotolos de Comunicação?

Protocolos de comunicação são conjuntos de regras e padrões que definem como os dados são enviados e recebidos em uma rede. Eles garantem que os dois dispositivos "falam a mesma língua". Sem protocolos, dispositivos diferentes não conseguiriam se comunicar. Exemplos incluem TCP/IP, HTTP, FTP e DNS, cada um com funções específicas no processo de comunicação em rede.

Modelo OSI

O Modelo OSI (Open Systems Interconnection) é um conceito de sete camadas que descreve como os dados viajam de uma aplicação em um dispositivo para uma aplicação em outro dispositivo através de uma rede. As camadas são:

  • Física - Meios de transmissão - fios, geralmente.
  • Enlace - Comunicação entre dispositivos da LAN
  • Rede - Roteamento de pacotes (Protocolo IP)
  • Transporte - Entrega confiável de dados (ex: TCP)
  • Sessão - Gerenciamento de conexões
  • Apresentação - Formatação de dados
  • Aplicação - interface com o usuário (ex: HTTP)

Cada camada tem responsabilidades definidas e comunica apenas com as camadas em cima e em baixo. O modelo OSI serve como referência para entender e melhor desenvolver protocolos de rede.

Modelo TCP/IP

O Modelo TCP/IP é o conjunto de protocolos que fundamenta a Internet e a maioria das redes modernas. É mais simples que o OSI, com apenas quatro camadas:

  • Acesso à Rede - Equivalente às camadas física e de enlace do OSI
  • Internet - Roteamento de pacotes (Protocolo IP)
  • Transporte - Entrega de dados (TCP é confiável, UDP é mais rápido mas não garente entrega,)
  • Aplicação -

Protocolo IPv4

O IPv4 (Internet Protocol version 4) é o protocolo responsável pelo endereçamento e roteamento de pacotes na Internet. Cada dispositivo recebe um endereço IP único composto por 32 bits (ex: 192.168.1.1). O IPv4 permite aproximadamente 4,3 bilhões de endereços únicos, número que com o crescimento muito rápido da Internet se tornou pouco.

Protocolo IPv6

O IPv6 (Internet Protocol version 6) é a versão mais recente do protocolo de Internet, desenvolvida para substituir o IPv4. Utiliza endereços de 128 bits (ex: 2001:0db8:85a3:0000:0000:8a2e:0370:7334), permitindo ~340 undecilhões de IPs únicos. Além de resolver o problema de poucos endereços, o IPv6 oferece melhorias em segurança, qualidade de serviço e eficiência. Atualmente estamos em transição, com os dois protocolos co-existindo.

Configuração TCP/IP com Node.js

Este código cria um servidor usando Node.js e Express. Ele define rotas para testar a simulaçãoo de um login.

Principais pontos

  • Configuração: O servidor roda na porta 3000 e no endereço localhost.
  • Middleware: Permite receber dados em JSON e habilita o uso de CORS.
  • Rota GET "/": Retorna uma mensagem dizendo que a aplicação está rodando.
  • Rota GET "/login": Retorna um usuário fixo (email e senha).
  • Rota POST "/login": Verifica se o email e senha enviados batem com os valores fixos:
    • Se corretos: retorna sucesso com nome e status.
    • Se senha errada: retorna "Acesso não autorizado".
    • Se email não encontrado: retorna "Usuário não encontrado".

const express = require('express')
const app = express()
const cors = require('cors')

// PORT do TCP
const PORT = 3000 
// endereço IP = 127.0.0.1 do servidor de teste
const hostname = 'localhost' 

// ------------ Middleware ---------------------
app.use(express.urlencoded({extended: true}))
app.use(express.json())
app.use(cors())

// ---------------------------------------------
app.get('/', (req,res)=>{
    res.status(200).json({message: 'Aplicação Rodando!'})
})

app.get('/login', (req,res)=>{
    const email_bd = 'carlos@gmail.com'
    const senha_bd = 123
    const dados = {
        email: email_bd,
        senha: senha_bd
    }
    res.status(200).json(dados)
})

app.post('/login', (req,res)=>{
    const email_bd = 'carlos@gmail.com'
    const senha_bd = 123
    const valores = req.body
    console.log(valores)
    
    if(email_bd === valores.email){
        if(senha_bd === valores.senha){
            return res.status(200).json({ 
                message: 'login realizado com sucesso!', 
                nome: valores.nome, 
                statusLog: 'true'
            })
        }else{
            return res.status(403).json({ 
                message: 'Acesso não autorizado!'
            })
        }
    }else{
        return res.status(404).json({
            message: 'Usuário não encontrado'
        })
    }
})

// ---------------- Server ---------------------
app.listen(PORT, hostname, ()=>{
    console.log(`http://${hostname}:${PORT}`)
})

Protocolo HTTP

O HTTP (HyperText Transfer Protocol) é o protocolo usado para transferir informações na Web. Ele define como as mensagens são formatadas e transmitidas entre navegadores (clientes) e servidores. O HTTP funciona no modelo requisição-resposta: o cliente envia uma requisição (GET, POST, PUT, DELETE, etc.) e o servidor responde com um código de status (200 OK, 404 Not Found, etc.) e os dados solicitados. É um protocolo sem estado, ou seja, cada requisição é independente da anterior.

Protocolo HTTPS

O HTTPS (HTTP Secure) é uma versão mais segura do HTTP, que adiciona uma camada de criptografia. Todos os dados transmitidos entre o navegador e o servidor são criptografados, protegendo informações sensíveis como senhas, dados de cartão de crédito e informações pessoais. O HTTPS também autentica o servidor através de certificados digitais (SLL), garantindo que o usuário está se comunicando com o site legítimo.

Express

Express é um framework para Node.js, utilizado para construir aplicações web e APIs. Ele simplifica o processo de criação de servidores HTTP, Com Express, é mais fácil definir rotas, gerenciar requisições e respostas, implementar middleware para processamento de dados, e organizar a estrutura da aplicação.

O método GET é usado para consultar dados no servidor sem modificá-los. Retorna status 200 quando bem-sucedido.

POST - Autenticação

app.post('/login', (req,res)=>{
    const email_bd = 'carlos@gmail.com'
    const senha_bd = 123
    const valores = req.body

    console.log(valores)

    if(email_bd === valores.email){
        if(senha_bd === valores.senha){
            return res.status(200).json({ 
                message: 'login realizado com sucesso!', 
                nome: valores.nome, 
                statusLog: 'true'
            })
        }else{
            return res.status(403).json({ 
                message: 'Acesso não autorizado!'
            })
        }
    }else{
        return res.status(404).json({
            message: 'Usuário não encontrado'
        })
    }
})

POST envia dados para o servidor processar. Retorna 200 (sucesso), 403 (não autorizado) ou 404 (não encontrado).
